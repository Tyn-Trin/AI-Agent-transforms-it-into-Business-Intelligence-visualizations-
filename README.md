# Odoo AI Chat Agent (Web)

**AI Agent ที่ให้ผู้ใช้งานในบริษัทถามข้อมูลจาก Odoo ERP เป็นภาษาคน (ไทย/อังกฤษ) ผ่านหน้าเว็บแชท แล้วได้คำตอบพร้อมกราฟวิเคราะห์ข้อมูลแบบ Real-time กลับมาทันที — พร้อมทำหน้าที่เป็น Support Assistant ในตัว ตอบคำถามวิธีใช้งาน Odoo ได้โดยไม่ต้องรอทีม Support**

ไม่ต้องรอรายงาน ไม่ต้องไล่หาข้อมูลในเมนู Odoo เอง ไม่ต้องรอคิวแชททีม Support — แค่ถาม แล้วให้ AI ตัดสินใจว่าจะดึงข้อมูลไหนมา หรือค้นหาวิธีทำจากคู่มือบริษัทมาตอบให้

## ปัญหาที่เจอ

ผู้ใช้งานที่ไม่ถนัดเทคนิค (ฝ่ายขาย จัดซื้อ บัญชี หรือผู้บริหาร) ต้องการคำตอบเร็วๆ จาก ERP เช่น "เดือนนี้ขายได้เท่าไหร่" "ลูกหนี้ค้างชำระเกินกำหนดมีใครบ้าง" แต่ติดปัญหา:
- ไม่รู้จัก filter, model หรือชื่อ field ของ Odoo
- ขอรายงานแบบกำหนดเองต้องรอ IT/BI ทำให้ เป็นวันๆ
- อยากเห็นเป็นกราฟเปรียบเทียบ/เทรนด์ ไม่ใช่แค่ตัวเลขก้อนเดียว

นอกจากเรื่องข้อมูล ยังมีอีกปัญหาที่เจอบ่อยไม่แพ้กัน — **ผู้ใช้เจอสถานการณ์ที่ไม่รู้ว่าต้องทำยังไง** ซึ่งมักไม่ใช่แค่ "ปุ่มอยู่ตรงไหน" (แบบนั้นใช้ไม่กี่ครั้งก็จำได้เอง) แต่เป็นความรู้เฉพาะของบริษัทที่ไม่มีเขียนไว้ที่ไหนนอกจากอยู่ในหัวรุ่นพี่หรือทีม Support เช่น:
- "ทำไมกด Confirm SO ไม่ได้ ทั้งที่สต็อกมีของ" (ต้อง Troubleshoot เช่น Credit Limit เกิน หรือ Pricelist ผิด)
- "ลูกค้าใหม่ขอ Credit Term เกิน 30 วัน ต้องขอใครอนุมัติ"
- "ของเสียหายระหว่างขนส่งต้องทำเอกสารอะไรบ้าง"
- "เอกสารมูลค่าเกิน 1 แสนบาทต้องผ่านใครอนุมัติบ้าง ตามลำดับไหน"

คำถามแบบนี้ต้องไปรอถามทีม Support ผ่านแชทอีกที กว่าจะมีคนว่างตอบก็เสียเวลาไปงานอื่นรอ

## ทางแก้

AI Agent (ใช้ Claude, Tool Use / Function Calling) ทำหน้าที่เป็นตัวกลางระหว่างผู้ใช้กับ Odoo ผู้ใช้พิมพ์คำถามเป็นภาษาคนผ่านหน้าเว็บแชท ตัว Agent จะตัดสินใจเองว่าต้องดึงข้อมูลอะไร เรียกใช้ Tool ที่ถูกต้องเพื่อดึงข้อมูลจาก Odoo แบบ Real-time แล้วตอบกลับเป็นภาษาคน — พร้อมแสดงกราฟแบบ interactive อัตโนมัติเมื่อข้อมูลนั้นเหมาะกับการทำกราฟ (อันดับ เทรนด์ เปรียบเทียบ)

และเพราะระบบผูกกับ **ChromaDB (Knowledge Base)** ที่เก็บคู่มือ/ขั้นตอนการทำงานของบริษัทไว้ AI จึงตอบคำถามเชิง How-to หรือ Troubleshoot เบื้องต้นได้ทันที เหมือนมี Support Team ส่วนตัวที่พร้อมตอบตลอดเวลา ไม่ต้องรอคิวแชทกับทีม Support จริง

**ขั้นตอนการทำงาน:**
1. ผู้ใช้ถามคำถามในหน้าเว็บแชท
2. คำถาม + ประวัติการสนทนาถูกส่งไปให้ Claude พร้อมชุด Tool ที่กำหนดไว้
3. Claude ตัดสินใจเองว่าต้องเรียก Tool ไหน — ออกแบบ Tool ไว้ไม่ให้โมเดลต้องเดาวันที่หรือตัวเลขเอง คำพูดที่บอกช่วงเวลาแบบสัมพัทธ์ ("เดือนนี้", "เดือนที่แล้ว") จะถูกคำนวณจริงฝั่ง Server จากวันที่ปัจจุบันจริง
4. ถ้าเป็นคำถามข้อมูล → Tool เรียก Odoo แบบ Real-time โดยใช้สิทธิ์ Odoo ของผู้ใช้คนนั้นเอง
5. ถ้าเป็นคำถามวิธีใช้งาน/ปัญหาที่เจอ → Tool ค้นหาจาก ChromaDB (Knowledge Base) ที่บริษัทอัพโหลดคู่มือไว้
6. Claude ตอบเป็นภาษาคน และถ้าข้อมูลเป็นอันดับ เทรนด์ หรือการเปรียบเทียบ หน้าเว็บแชทจะแสดงกราฟ interactive คู่กับคำตอบให้อัตโนมัติ — กราฟจะถูกบันทึกไว้ด้วย เปิดประวัติแชทเก่ากลับมาดูก็ยังเห็นกราฟเดิม

## Tech Stack

| ส่วน | เทคโนโลยี |
|---|---|
| Backend | Python 3.x, FastAPI, Uvicorn |
| AI Reasoning | Anthropic Claude (Tool Use / Function Calling) |
| แหล่งข้อมูล ERP | Odoo JSON-RPC (ดึงแบบ Real-time ไม่ต้องเก็บข้อมูลซ้ำ) |
| แสดงผลกราฟ | Plotly (แสดงในหน้าเว็บแชท) |
| Database | PostgreSQL |
| Knowledge Base | ChromaDB (RAG สำหรับคู่มือ/How-to/นโยบายบริษัท) |

## Feature เด่น

- **ถามเป็นภาษาคนได้เลย** — ไม่ต้องรู้จัก filter, model หรือชื่อ field ของ Odoo
- **ข้อมูล Real-time** — ทุกคำตอบดึงจาก Odoo จริงตามสิทธิ์ของผู้ใช้คนนั้น
- **เป็น Support Assistant ในตัว** — ถามเรื่อง Troubleshoot ("ทำไม Confirm SO ไม่ได้"), นโยบายบริษัท ("Credit Term เกิน 30 วันขอใคร"), หรือขั้นตอนกรณีพิเศษ ("ของเสียหายระหว่างขนส่งทำเอกสารอะไร") ตอบได้ทันทีจากคู่มือที่บริษัทใส่ไว้ ไม่ต้องรอทีม Support ตอบแชท
- **กราฟแบบ BI ตามคำขอ** — อันดับ (top customer/product/vendor), เทรนด์, เปรียบเทียบช่วงเวลา แสดงเป็นกราฟ interactive อัตโนมัติ ไม่ต้องสร้าง dashboard เอง
- **คำนวณช่วงเวลาฝั่ง Server** — "เดือนนี้", "ไตรมาสที่แล้ว", "ปีที่แล้ว" ฯลฯ คำนวณจากวันที่จริงบน Server เสมอ ไม่ให้ AI เดาเอาเอง คำตอบจึงตรงช่วงเวลาจริงทุกครั้ง ไม่ hallucinate
- **กราฟ Persist** — เปิดบทสนทนาเก่ากลับมา กราฟยังอยู่เหมือนเดิม
- **รองรับ 2 ภาษา (ไทย/อังกฤษ)**
